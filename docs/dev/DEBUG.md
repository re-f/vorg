# VOrg 调试指南

本指南帮助你诊断和解决使用 VOrg 扩展时遇到的常见问题。

## 快速诊断

根据你遇到的问题类型，跳转到相应章节：

- [查询结果不符合预期](#1-查询相关问题) → 使用 VOrgQL 调试工具
- [编辑命令不生效](#2-编辑命令问题) → 检查命令注册和上下文
- [语法高亮显示错误](#3-语法高亮问题) → 刷新高亮或检查语法定义
- [文件未被索引](#4-文件索引问题) → 检查数据库和文件监听
- [扩展响应缓慢](#5-性能问题) → 性能分析和优化建议

## 1. 查询相关问题

### 症状
- 查询结果缺少预期的任务或标题
- 某些文件或标签没有被搜索到
- `group-by` 分组结果不正确

### 调试工具：VOrgQL 命令行调试器

```bash
npx ts-node src/test/unit/debug_vorgql_real.ts \
  --dir ~/your-org-files \
  --query "(and (todo \"TODO\") (tag \"work\"))"
```

**工具功能**：
- 扫描指定目录并建立临时索引
- 显示查询翻译成的 SQL 语句和参数
- 列出所有匹配结果
- **差异分析**：对比数据库全量标签与查询结果，标记缺失数据

### 调试步骤

1. **验证文件是否被扫描**
   ```bash
   npx ts-node src/test/unit/debug_vorgql_real.ts --dir . --query "(todo)"
   ```
   检查输出中的 "Found X .org files" 数量是否正确

2. **检查 SQL 翻译**
   查看输出中的 "SQL WHERE" 和 "Params" 部分，确认查询逻辑是否符合预期

3. **使用差异分析**
   对于标签查询，工具会自动显示数据库中存在但查询未命中的标签（`[WARNING]`）

4. **自定义 TODO 关键词**
   如果你使用了自定义关键词配置：
   ```bash
   --todo "TODO NEXT | DONE CANCELLED"
   ```

### 常见问题

**Q: 为什么某个标签搜不到？**
- 检查标签格式是否正确（`:tag:`）
- 使用调试工具查看数据库中的标签列表
- 确认文件已保存并被索引

**Q: 优先级查询不工作？**
- VOrg 使用 `[#A]` 格式存储优先级
- 查询时使用 `(priority "A")` 即可，系统会自动匹配

## 2. 编辑命令问题

### 症状
- 快捷键按下后没有反应
- 编辑操作行为异常（如插入位置错误）
- 光标跳转到错误位置

### 调试步骤

#### 2.1 检查命令是否注册

1. 打开命令面板（`Cmd+Shift+P` / `Ctrl+Shift+P`）
2. 输入 "VOrg" 查看所有可用命令
3. 确认你期望的命令是否在列表中

#### 2.2 查看输出日志

1. 打开 **输出** 面板（`View > Output`）
2. 在下拉菜单中选择 "VOrg"
3. 执行问题命令，观察日志输出

#### 2.3 使用断点调试（开发者）

1. 按 `F5` 启动扩展开发宿主
2. 在 `src/commands/` 相关文件中设置断点
3. 在开发宿主窗口中触发命令
4. 在调试控制台查看变量值

### 常见问题

**Q: Meta-Return 不插入新标题？**
- 检查光标是否在 `.org` 文件中
- 确认当前行的上下文（使用 `ContextAnalyzer`）
- 查看输出日志中的错误信息

**Q: 属性抽屉编辑后格式错乱？**
- 检查 `PropertyParser` 的解析结果
- 确认抽屉的开始和结束标记是否完整

## 3. 语法高亮问题

### 症状
- 关键词颜色不正确
- 代码块或表格高亮异常
- 链接未被识别

### 调试步骤

#### 3.1 手动刷新高亮

1. 打开命令面板
2. 执行 `VOrg: Refresh Syntax Highlighting`
3. 观察高亮是否恢复正常

#### 3.2 检查语法定义

查看 `syntaxes/org.tmLanguage.json` 中的正则表达式是否正确匹配你的内容。

#### 3.3 检查主题兼容性

某些 VS Code 主题可能不支持 Org-mode 的特定 token，尝试切换主题测试。

### 常见问题

**Q: TODO 关键词没有高亮？**
- 确认关键词在 `settings.json` 中正确配置
- 检查关键词是否在标题行的正确位置（星号之后，标题文本之前）

## 4. 文件索引问题

### 症状
- 新创建的文件不出现在查询结果中
- 修改文件后查询结果未更新
- 数据库文件 `.vorg.db` 不存在或过时

### 调试步骤

#### 4.1 检查数据库文件

在工作区根目录查找 `.vorg.db` 文件：
```bash
ls -la .vorg.db
```

#### 4.2 查看数据库内容（需要 SQLite 工具）

```bash
sqlite3 .vorg.db "SELECT COUNT(*) FROM files;"
sqlite3 .vorg.db "SELECT COUNT(*) FROM headings;"
```

#### 4.3 强制重新索引

1. 删除 `.vorg.db` 文件
2. 重新加载 VS Code 窗口（`Cmd+R` / `Ctrl+R`）
3. VOrg 会自动重建索引

### 常见问题

**Q: 为什么有些文件没被索引？**
- 检查文件扩展名是否为 `.org`
- 确认文件在工作区目录内
- 查看输出日志中的文件扫描信息

**Q: 修改文件后查询结果没变化？**
- 确保文件已保存（`Cmd+S` / `Ctrl+S`）
- 检查文件监听是否正常工作
- 尝试手动触发索引更新

## 5. 性能问题

### 症状
- 打开大型 `.org` 文件时卡顿
- 查询响应缓慢
- CPU 占用率高

### 调试步骤

#### 5.1 使用 VS Code 性能分析器

1. 打开命令面板
2. 执行 `Developer: Show Running Extensions`
3. 查看 VOrg 的激活时间和内存占用

#### 5.2 检查文件大小和数量

```bash
find . -name "*.org" -exec wc -l {} + | sort -n
```

#### 5.3 优化建议

- **大文件**：考虑拆分为多个小文件
- **大量文件**：使用 `.vorgignore` 排除不需要索引的目录
- **复杂查询**：简化查询条件，避免过多的 `OR` 组合

### 常见问题

**Q: 为什么索引很慢？**
- 检查工作区中 `.org` 文件的总数和大小
- 确认没有索引 `node_modules` 等无关目录
- 考虑使用增量索引而非全量重建

## 开发者调试

### 运行单元测试

```bash
pnpm run test:unit
```

### 运行集成测试

```bash
pnpm run test
```

### 启动扩展开发宿主

按 `F5` 或在调试面板选择 "Run Extension"

### 查看扩展日志

所有日志输出到 VS Code 的 "输出" 面板，选择 "VOrg" 频道。
