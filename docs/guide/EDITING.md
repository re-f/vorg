# 高效编辑指南 (Editing Guide)

本指南介绍 VOrg 在 VS Code 中复现的 Emacs Org-mode 编辑特性，包括智能插入、折叠控制、属性管理与结构调整。

### M-RET: 上下文感知的“万能回车”
快捷键：`Alt+Enter` (Win/Linux) / `Opt+Enter` (Mac)

VOrg 会根据光标所在的位置，智能判断该插入什么：

*   **在标题上**：插入同级标题。
*   **在列表项中**：插入同级列表项。
*   **在 Checkbox 列表中**：插入带 `[ ]` 的新任务项。
*   **在表格行中**：插入新行并自动对齐边框。
*   **在普通文本中**：将当前行转换为标题。

> [!TIP]
> **Shift+Alt+Enter**: 快速插入一个新的 **TODO** 标题。

### C-RET: 结构化插入与分割
快捷键：`Ctrl+Enter` (Win/Linux/Mac)

*   **当前子树末尾插入**：当您在编辑一个长篇大论的子条目时，无需手动滚动到底部，直接按 `Ctrl+Enter`，即可在当前子树的**最后**插入一个同级标题。
*   **智能分割**：如果在行中间按下，它会将光标后的内容切割并作为新标题的内容。

## 2. 智能折叠与可视性 (Smart Folding)

快捷键：`Tab` / `Shift+Tab`

VOrg 还原了 Org-mode 经典的 TAB 循环机制：

1.  **全局折叠 (Global)**: 折叠所有内容，只看一级大纲。
2.  **内容查看 (Contents)**: 展开所有标题，但隐藏具体正文。
3.  **全部展开 (Show All)**: 展开所有细节。

**智能上下文支持**：
*   **在标题上**：折叠/展开该子树。
*   **在 Property Drawer 上**：折叠属性抽屉，保持文档整洁。
*   **在代码块头 (`#+BEGIN_SRC`)**：折叠代码块，只看逻辑。
*   **在表格中**：Tab 键变为“跳转到下一单元格”，Shift+Tab 为“上一单元格”。

## 3. 属性与元数据管理 (Property Management)

为您的笔记添加结构化数据，是构建“第二大脑”的基础。

### 快捷设置
快捷键：`Ctrl+C Ctrl+X P`

无需手动编写 `:PROPERTIES:` 抽屉：
1.  在任意标题上按下快捷键。
2.  输入属性名（如 `CATEGORY`, `PRIORITY`）。
3.  输入属性值。

**VOrg 会自动**：
*   创建符合规范的 `:PROPERTIES:` ... `:END:` 抽屉。
*   如果已有抽屉，自动插入并对齐缩进。
*   如果这是首次设置，会自动生成一个用于全局跳转的 **UUID**。

## 4. 结构化操作 (Structure Editing)

像操作积木一样操作文档结构，而不是简单的文本复制粘贴。

*   **升级子树 (Promote Subtree)**: `Ctrl+C Ctrl+Shift+,` (对应 `<`)
    *   将当前标题及其**所有子节点**提升一级（例如：三级变二级）。
*   **降级子树 (Demote Subtree)**: `Ctrl+C Ctrl+Shift+.` (对应 `>`)
    *   将当前标题及其**所有子节点**降低一级。

> [!NOTE]
> 所有的层级调整都会智能保持子树内部的相对结构不变。

## 5. 自动补全与链接 (Completion)

*   **ID 链接补全**：输入 `[[` 或 `[[id:`，VOrg 会扫描整个工作区，提供所有标题的模糊搜索列表，选择后插入 `[[id:UUID][标题]]`；若目标标题尚无 ID，会自动写入 `:ID:` 属性。
*   **插入标题链接（命令）**：按 `Ctrl+C Ctrl+L` 选择 **Heading**，与补全共用同一套逻辑——跨文件选标题、生成 `id:` 链接、自动补 ID。
*   **链接跳转**：`Ctrl+C Ctrl+O`（跟随链接）、Ctrl+Click / F12 使用同一套解析逻辑，支持 `id:`、纯标题、`*标题`、`file:path` 及 `file:path::*标题` / `file:path::#id` 锚点。
