## 任务目标
为 `refile` 命令补上真实 VS Code 环境下的集成测试，验证命令接线、交互和文档编辑结果。

## 本任务必须遵守
- 先读 `PRINCIPLES.md`
- 集成测试主要验证命令行为，不重复覆盖纯逻辑细节
- 纯逻辑规则优先放在单元测试里

## 建议文件
- `src/test/suite/refileCommands.test.ts`

## 优先覆盖的场景
- 当前标题 refile 到另一个一级标题下，应变成二级标题
- 带子标题的 subtree 一起移动，且相对层级保持不变
- refile 后源位置被删除
- 重名标题通过 outline path 仍能选对
- 目标是 source 子树内部时应拒绝执行

## 建议增加的交互场景
- 用户取消 Quick Pick，不应修改文档
- 没有合法目标时，不应修改文档
- 光标不在 headline 上时，不应修改文档

## 重点检查
- 最终文档文本是否正确
- 光标或可见状态是否需要最小化处理
- 命令执行后不会产生重复 subtree 或残留旧内容

## 验收点
- 在真实扩展测试宿主中，至少有一条成功路径用例验证：source subtree 从原位置消失，并在目标 headline 下以正确层级重新出现。
- 至少有一条集成测试覆盖带子标题的 subtree 整体移动，并断言移动后子标题相对层级保持不变。
- 至少有一条集成测试覆盖重名 headline，通过 outline path 选择正确目标，并断言 subtree 挂到了正确父节点下。
- 至少有一条集成测试覆盖“目标位于 source 子树内部”的非法场景，并断言执行前后文档全文一致。
- 至少分别覆盖“用户取消 Quick Pick”“无合法目标”“光标不在 headline 上”三类否定路径，并断言文档无改动。

## 完成标准
- 关键命令路径已有集成测试覆盖
- 集成测试与单元测试分工清晰
- 失败时能明确指出是命令层接线问题还是 planner 逻辑问题
